Basico de Git

Iniciar un proyecto

Para poder inicializar un proyecto con Git instalado simplemente debes de colocar el siguiente comando 

git init

recuerda que debes de estar en el directorio donde haras tu proyecto

Agregar archivos a preparacion

Pasar uno o varios archivos de working a preparacion con el siguiente comando 

 git add archivo.txt

si quieres que todos los archivos se suban de one, colocas un punto al final 

git add .

Ver archivos los cuales seran commiteados o no STATUS

Visualizar cuando vamos a hacer un commit es importante, y debemos saber que cosas tanto del working como del preparacion van a ser comiteadas o no 

 git status

Quitar archivos de preparacion

eliminar uno o varios archivos de preparacioncon el siguiente comando 

 git rm --cached archivo.txt

Llevar de preparacion a commited

Pasar uno o varios archivos de working a preparacion con el siguiente comando 

 git commit -m "mensaje de referencia"

Commit Instantaneo

Si quieres que tus archivos no pasen por el area de preparacion puedes commitearlos directamente del area de working hacia commited 

git commit -a

Eliminar un archivo de commited

tendremos que eliminar primero el archivo en working luego hacer un add y enviarlo a preparacion luego hacerle un commit para que de esta manera el repositorio o commited se le notifique que como ya no esta en el directorio lo eliminara el tambien. 

  • rm archivo.txt
  • git status

    te saldra que el archivo es deleted 

  • git add archivo.txt
  • git commit -m"Notificando que el archivo se elimino"

Recuperar archivo eliminado o midificado de working del area de preparacion

si subiste un archivo al area de preparacion y luego ese mismo archivo, lo modificas o eliminas de tu directorio o tambien llamado working lo puedes recuperar con el siguiente comando 

git restore archivo.txt

Volver al ultimo commit que hiciste de un archivo en working

si subiste un archivo al commited y despues hiciste cambios y de los cuales no te gustaron, puedes volver al tiempo y usar: 

git checkout archivo.txt

te devuelve el ultimo archivo comiteado que hiciste al area de working y remplaza ese archivo modificado

ADVERTENCIA !

Este comando solo es funcionable cuando no haz hecho un add, por el contrario si ya subiste ese archivo modificado al area de preparacion no te funcionara, ten lo en cuenta, si en caso lo hiciste quieres aun asi otra solucion ten en cuenta la siguiente recomendaci√≥n de RESET

RESETEAR todo el proyecto

si por alguna razon haz hecho mal los pasos, y te gustaria volver al ultimo commit pero como si literalmente no hubieras hecho nada puedes usar 

git reset --hard

usalo con cuidado ya que literalmente es como viajar en el tiempo a tu ultimo commit

Cambiar nombre de archivo que esta en commited

si por alguna razon cambiar el nombre de un archivo, coloquemos lo siguiente: 

git mv archivo.txt nuevoNombre.txt

Luego simplemente lo llevamos al area de preparacion y lo comiteamos y automaticamente se cambiara el mismo archivo pero con otro nombre dentro de commited

Visualizar archivos del area de commited

te permite ver en la terminal de git el documento que hiciste ultimo commit 

git show archivo.txt

Ver diferencia entre el area de preparacion y commited

Muchas veces queremos ver que tenemos en el area de preparacion y commited y verlas al mismo tiempo para ver que diferencias tienen para ese utiliza 

git diff --staged

Intermedio de Git

Identificar Commits de un archivo commited

Si desea saber las diferencias entre commits que hiciste de un archivo ya sea que hayas hecho 10 commits de ese archivo, debes de tener en cuenta que cada commit tiene un ID por asi decirlo que se hace asi : 

git log

para que sea mas simplificado usa

git log --oneline
