Basico de Git

Iniciar un proyecto

Para poder inicializar un proyecto con Git instalado simplemente debes de colocar el siguiente comando 

git init

recuerda que debes de estar en el directorio donde haras tu proyecto

Agregar archivos a preparacion

Pasar uno o varios archivos de working a preparacion con el siguiente comando 

 git add archivo.txt

si quieres que todos los archivos se suban de one, colocas un punto al final 

git add .

Ver archivos los cuales seran commiteados o no STATUS

Visualizar cuando vamos a hacer un commit es importante, y debemos saber que cosas tanto del working como del preparacion van a ser comiteadas o no 

 git status

Quitar archivos de preparacion

eliminar uno o varios archivos de preparacioncon el siguiente comando 

 git rm --cached archivo.txt

Llevar de preparacion a commited

Pasar uno o varios archivos de working a preparacion con el siguiente comando 

 git commit -m "mensaje de referencia"

Commit Instantaneo

Si quieres que tus archivos no pasen por el area de preparacion puedes commitearlos directamente del area de working hacia commited 

git commit -a

Eliminar un archivo de commited

tendremos que eliminar primero el archivo en working luego hacer un add y enviarlo a preparacion luego hacerle un commit para que de esta manera el repositorio o commited se le notifique que como ya no esta en el directorio lo eliminara el tambien. 

  • rm archivo.txt
  • git status

    te saldra que el archivo es deleted 

  • git add archivo.txt
  • git commit -m"Notificando que el archivo se elimino"

Recuperar archivo eliminado o midificado de working del area de preparacion

si subiste un archivo al area de preparacion y luego ese mismo archivo, lo modificas o eliminas de tu directorio o tambien llamado working lo puedes recuperar con el siguiente comando 

git restore archivo.txt

Volver al ultimo commit que hiciste de un archivo en working

si subiste un archivo al commited y despues hiciste cambios y de los cuales no te gustaron, puedes volver al tiempo y usar: 

git checkout archivo.txt

te devuelve el ultimo archivo comiteado que hiciste al area de working y remplaza ese archivo modificado

ADVERTENCIA !

Este comando solo es funcionable cuando no haz hecho un add, por el contrario si ya subiste ese archivo modificado al area de preparacion no te funcionara, ten lo en cuenta, si en caso lo hiciste quieres aun asi otra solucion ten en cuenta la siguiente recomendación de RESET

RESETEAR todo el proyecto

si por alguna razon haz hecho mal los pasos, y te gustaria volver al ultimo commit pero como si literalmente no hubieras hecho nada puedes usar 

git reset --hard

usalo con cuidado ya que literalmente es como viajar en el tiempo a tu ultimo commit

Cambiar nombre de archivo que esta en commited

si por alguna razon cambiar el nombre de un archivo, coloquemos lo siguiente: 

git mv archivo.txt nuevoNombre.txt

Luego simplemente lo llevamos al area de preparacion y lo comiteamos y automaticamente se cambiara el mismo archivo pero con otro nombre dentro de commited

Visualizar archivos del area de commited

te permite ver en la terminal de git el documento que hiciste ultimo commit 

git show archivo.txt

Ver diferencia entre el area de preparacion y commited

Muchas veces queremos ver que tenemos en el area de preparacion y commited y verlas al mismo tiempo para ver que diferencias tienen para ese utiliza 

git diff --staged

Intermedio de Git

Identificar Commits de un archivo commited

Si desea saber las diferencias entre commits que hiciste de un archivo ya sea que hayas hecho 10 commits de ese archivo, debes de tener en cuenta que cada commit tiene un ID por asi decirlo que se hace asi : 

git log

para que sea mas simplificado usa

git log --oneline

Hacer identificadores Pequeños a los commited

existe una probabilidad de que las id de los commits que se crean, puedan ser iguales por lo cual es mejor tener un numero de caracteres fijo dependiendo de que tan grande vayas a hacer un proyecto con commits 

git config --global core.abbrev 5

en este caso el numero que puse "5" puede variar dependiendo de cuantos commits hagas y del proyecto, mientras mas grande sea mejor seria colocarle un numero mas grande

Comparar entre dos commited

Una vez ya sabido los dos pasos anteriores, ahora si podemos comparar dos commits, para que por asi decirlo ver que cambiamos de un archivo de un antes y despues de los commits que hayamos hecho, ayudandonos con los identificadores 

git diff 4427b 9ccc3

los numeros son referenciales, independientemente de los id que tengas de los commits que hayas hecho

Otras cosas de DIFF

puedes ver directamente que archivos cambiaron con:

git diff --name-only 4427b 9ccc3

si quieres ver a nivel lineas que cambio puedes usar

git diff --word-diff 4427b 9ccc3

Modificar Commit

En si no se puede modificar un commit pero si se puede remplazar. 

git commit --amend

lo que hace es cargar el ultimo commit y te permite cambiar la descripcion del commit, ahora digamos que quieres subir un archivo y modificar archivos de ese commit que decidiste modificar, lo que debes hacer es antes de hacer git commit --amendl, subirlo al area de preparacion y ya luego ahi recien subirlo con el codigo que proporcione primero.

Que es HEAD?

Te preguntas que es eso de HEAD? cuando haces un

git log

pues bueno para que entiendas como funciona te lo describo con la siguiente secuencia

A
B
C
D

El ultimoque es head es el ultimo commit que hemos hecho si es la primera vezq ue hacemos un commit pues solo habra un circulito que serai a, si hacemos otro commit tendremos 2 circulitos , si hacemos 3 .... y asi sucesivamente y HEAD siempre va a estar siempre al ultimo

ACLARACION DE MODIFICAR LOS COMMITS

Cuando intentes hacer una modificacion de commit con ammend, recuerda no es como que entramos a ese commit y modificamos literalmente, sino que agarramos el commit lo modificamos y es remplazado como el nuevo ultimo commit y ya no el que teniamos antes , es decir si por ejemplo el utlimo commit que hicimos en E como en la imagen de los circulos, entonces digamos que quiero modificar algo, lo hago con amend entonces ese E sera remplazado por otro E y el anterior E sera eliminado, por lo tanto tambien cambiara su ID

Ahora si quisieras hacer una "MODIFICACION" de otros commits atras, ya no te lo recomiendo ya que es un proceso mas compliado, lo normal es asegurarte de que haces los commits bien y que si fallas solo sea en el ultimo commit ya que es mas viable que modificar otros commits anteriores.
